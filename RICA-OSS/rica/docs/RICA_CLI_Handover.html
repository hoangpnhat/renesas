Overview

What is RICA CLI?

A rebranded overlay of Continue CLI using the same shim pattern as the VS Code extension. RICA-specific logic lives in rica/cli/ and is injected at build time.

Authentication

Token Exchange Flow

rica login → LLM request
// 1. Login: user pastes API key rica login → POST /api/extension/token Headers: { "X-API-Key": rc_... } → Response: { access_token: "<MDP_TOKEN>" } → Save API key to ~/.rica/config.json // 2. Every LLM request: fresh token exchange ensureValidToken() → if cached && !expired: return cached → else: exchangeApiKeyForToken(apiKey) → Bearer <MDP_TOKEN> for Databricks

Security: MDP tokens are never persisted to disk. Memory-only, exchanged fresh each session.

Design

Key Decisions

Build-time shims over runtime patches — Zero upstream merge conflicts
API key auth (not EntraID) — No browser needed, works in headless/SSH
MDP token never on disk — Security constraint, fresh exchange each session
PostHog disabled — CLI does not send telemetry to external services
Same brand.json source — Version, name, URLs shared with VS Code extension

Issues

Known Issues

Login 401 on production — /api/extension/token may not accept X-API-Key header in prod. Works on staging.
No config.yaml template — First-time users have no model config. Need template generation on login.
Bundle size 14 MB — Includes stubbed core/ modules (embeddings, indexing). Could tree-shake further.
Dynamic config disabled — RICA API not ready for CLI. Fallback to hardcoded default model.
